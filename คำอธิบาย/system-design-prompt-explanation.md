# ทำความรู้จักโปรเจกต์ System Design Prompt

โปรเจกต์ **System Design Prompt** เป็นเว็บแอปพลิเคชัน (Web Application) ที่ถูกสร้างขึ้นมาเพื่อช่วยผู้ใช้งานในการสร้าง "คำสั่ง (Prompt)" สำหรับนำไปใช้พูดคุยกับ AI Models ยอดนิยม (เช่น ChatGPT, Claude, Gemini, GitHub Copilot) เพื่อช่วยในการออกแบบสถาปัตยกรรมระบบ (System Architecture) ให้เหมาะสมกับบริบทและความต้องการของแต่ละแผนกในองค์กร

## โครงสร้างของโปรเจกต์ (Project Structure)
โปรเจกต์นี้มีการออกแบบสถาปัตยกรรมแบบ Client-Server โดยแบ่งการทำงานออกเป็น 2 ส่วนหลัก ได้แก่ **Frontend (หน้าบ้าน)** และ **Backend (หลังบ้าน)**

### 1. Backend (ส่วนหลังบ้าน)
ถูกพัฒนาขึ้นด้วย **Node.js** ร่วมกับเฟรมเวิร์ก **Express.js** 
- **ไฟล์หลักในการทำงาน:** `backend/server.js`
- **หน้าที่หลัก:** ทำหน้าที่เป็น RESTful API Server เพื่อรองรับการร้องขอข้อมูลจากฝั่ง Frontend และเป็นตัวประมวลผลเพื่อสร้าง Prompt กลับไป
- **API Endpoints (ช่องทางการเชื่อมต่อ):**
  - `GET /api/options`: มีหน้าที่ส่งข้อมูลพื้นฐานสำหรับการสร้างฟอร์ม (Dropdown options) ไปให้ฝั่ง Frontend ซึ่งประกอบด้วย **รายชื่อ AI Models** และ **รายชื่อแผนกต่างๆ** (เช่น IT, HR, Marketing, Sales, Finance, Operations)
  - `POST /api/generate-prompt`: เป็น API สำหรับรับข้อมูลที่ผู้ใช้กรอกจาก Frontend (ได้แก่ AI Model ที่เลือก, แผนกที่เลือก, และบริบทของโปรเจกต์) นำมาประกอบร่างรวมกับ **Prompt Template** ที่เตรียมไว้ในระบบ เพื่อสร้างคำสั่ง Prompt ที่มีความสมบูรณ์ และมีโครงสร้างหัวข้อชัดเจน ก่อนจะส่งกลับไปให้ผู้ใช้งาน

### 2. Frontend (ส่วนหน้าบ้าน)
ถูกพัฒนาด้วยไลบรารี **React.js** (จัดการและสร้างโปรเจกต์ผ่านเครื่องมือ **Vite**)
- **ไฟล์หลักในการทำงาน:** จะอยู่ในโฟลเดอร์ `frontend/src` ซึ่งประกอบด้วยไฟล์หน้าจอหลักและ Components ต่างๆ (เช่น `App.jsx`, `PromptForm.jsx`, `PromptResult.jsx`)
- **หน้าที่หลัก:**
  - สร้างส่วนติดต่อผู้ใช้งาน (User Interface - UI) เพื่อให้ผู้ใช้งานสามารถทำรายการได้ง่าย
  - ติดต่อกับ Backend เพื่อดึงข้อมูลตัวเลือก (Options) มาแสดงในฟอร์มได้อย่างอัตโนมัติ
  - ให้ผู้ใช้สามารถกรอกข้อมูล "รายละเอียดของโปรเจกต์ (Project Context)"
  - รับข้อมูลจากผู้ใช้ ส่งผ่าน API ไปยัง Backend และนำผลลัพธ์ (Prompt ที่ถูกสร้างเสร็จแล้ว) กลับมาแสดงผลให้ผู้ใช้สามารถคัดลอก (Copy) ไปใช้งานต่อได้สะดวก

## การทำงานโดยภาพรวม (How it works)
1. **เข้าสู่เว็บไซต์:** ผู้ใช้งานเปิดหน้าเว็บขึ้นมา Frontend จะทำการเรียก `GET /api/options` เพื่อดึงข้อมูลรายชื่อ AI และรายชื่อแผนก มาแสดงเป็นตัวเลือกให้ผู้ใช้
2. **กรอกข้อมูล:** ผู้ใช้งานทำการเลือก AI Model (เช่น เลือก Gemini), เลือกแผนก (เช่น เลือก HR), และพิมพ์รายละเอียดความต้องการลงในช่อง Project Context
3. **ส่งข้อมูล:** เมื่อผู้ใช้งานกดปุ่มสร้าง Prompt ตัว Frontend จะส่งข้อมูลทั้งหมดไปยัง Backend ผ่าน `POST /api/generate-prompt`
4. **ประมวลผล:** Backend จะรับข้อมูลเหล่านั้นไปแทรกในข้อความ Template เพื่อสั่งให้ AI สวมบทบาทเป็นสถาปนิกระบบที่เชี่ยวชาญในสายงานนั้นๆ (Role-playing) โดยกำหนดให้ผลลัพธ์ต้องตอบครอบคลุม 5 หัวข้อหลัก (Tech Stack, Core Features, Security, Deployment เป็นต้น)
5. **แสดงผลลัพธ์:** Backend ส่งข้อความ Prompt ที่ประกอบเสร็จแล้วกลับมา และ Frontend นำมาแสดงบนหน้าจอให้ผู้ใช้นำไปใช้งานกับ AI ตัวที่ต้องการได้ทันที

## สรุป (Conclusion)
โปรเจกต์ "System Design Prompt" นี้เปรียบเสมือนเครื่องมือช่วยทุ่นแรง (Utility Tool) สำหรับนักพัฒนาหรือผู้จัดการโปรเจกต์ ที่ต้องการไอเดียในการวางโครงสร้างระบบ ด้วยการดึงความสามารถของ Generative AI ออกมาใช้อย่างเต็มประสิทธิภาพผ่านโครงสร้าง Prompt ที่ผ่านการคิดมาอย่างดี (Prompt Engineering) ทำให้ได้ผลลัพธ์ที่มีความเป็นมืออาชีพ ครอบคลุมทั้งด้านโครงสร้าง ความปลอดภัย และการนำระบบไปติดตั้งจริง
